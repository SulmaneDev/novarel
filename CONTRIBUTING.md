# Contributing to Novarel

Thank you for your interest in contributing to Novarel! To ensure a smooth collaboration, please follow these guidelines when submitting your contributions.

## How to Contribute

### 1. Fork the Repository

Start by forking the official Novarel repository to your own GitHub account.

### 2. Create a Branch

Create a new branch for your changes within your fork. Use a clear and descriptive naming convention:

- For new features: `feature/your-feature-name`
- For bug fixes or error solutions: `fix/your-fix-name` or `solution/issue-description`

### 3. Implement Your Changes

Write clean, maintainable code that follows the project's standards. Ensure any new functionality is well-documented.

### 4. Commit and Push

Provide clear and descriptive commit messages that explain the _what_ and _why_ of your changes.

```bash
git push origin feature/your-feature-name
```

### 5. Submit a Pull Request

When you are ready, submit a Pull Request (PR) from your branch to the `main` branch of the Novarel repository.

#### Pull Request Requirements:

- **Clear & Advanced Title**: Use a concise title that summarizes the change (e.g., `feat(core): implement automatic dependency injection via constructor`).
- **Detailed Description**: Provide a standard description that includes:
    - **What**: A clear explanation of the changes made.
    - **Why**: The rationale behind the implementation or the problem it solves.
    - **Impact**: Any potential side effects or breaking changes.

---

We appreciate your contributions to making Novarel a better framework!
